# Study Offline — How to Use This Repo With No Internet

Everything you need is in this repo. No LeetCode, no blogs, no Google required. This page is the map.

## What's here

## The daily loop (study weeks, W1–W5)

1. Open the day file, e.g. `prep-weeks/week-01/01-mon-jun-29.md`. Read its 📌 *Study today*.
2. **Block A (DSA):** read the theory + worked example. Then attempt each problem in the practice set **before** looking — open the matching pattern file in `dsa-solutions/` to check the full statement + solution.
3. **Block B (core topic):** read it; if you want the fuller treatment, jump to the linked module (`01`–`08`).
4. **Block C:** second DSA set / deep-dive.
5. Answer the day's 🎤 interview questions aloud, then check `week-0N/interview-answers.md`.
6. Tick `PROGRESS.md`. Sunday: run the week README's self-assessment gate — don't advance if you fail it.

## The execution loop (W6–W12)

Open the day file, do the activity (apply batch / OA drill / loop prep / negotiation step), and log it in `applications-and-referrals.md`. Keep the **Floor / Safety-Net ladder** running in parallel from Week 6 — it's the "employed no matter what" insurance.

## Fast revision (any time)

- `cheat-sheets/dsa-patterns.md` — recognize the pattern + recall the template + Big-O.
- `cheat-sheets/system-design.md` — the framework + design one-pagers + estimation numbers.
- `cheat-sheets/java-spring.md` — collections/concurrency/JVM/Spring one-liners.
- `projects-reference.md` — your three projects, cold.

## The night before an interview

`projects-reference.md` (your stories) → the relevant `cheat-sheets/*` → the company-tagged practice in `dsa-solutions/` → skim `interview-qa.md`. Sleep.
