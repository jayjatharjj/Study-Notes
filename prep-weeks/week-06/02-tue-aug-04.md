# Week 6 · Day 2 — Tue Aug 04 — ATS Tailoring Routine + Application Batch

📌 **Today:** Lock in the 60-second per-application ATS-tailoring reflex, push the running total to 8–10 applications, and memorise the CTC + availability answers before the first screen lands.

**DSA maintenance (~45 min):** Two easy hashing reps:
- [ ] **LC 217 – Contains Duplicate** (Easy) — HashSet. Under 5 min.
- [ ] **LC 242 – Valid Anagram** (Easy) — frequency array. Under 8 min. State why a 26-int array beats a HashMap here.

**Primary activity:**

*The ATS-tailoring routine (~45 min) — run on every application:*

Most sub-20 LPA applications are filtered by an automated ATS *before* a human sees them. A resume that doesn't echo the JD's exact terms gets dropped silently. Make this a 60-second reflex:
- [ ] Keep a **base resume** + produce a tailored variant per JD in 60 seconds.
- [ ] Paste the JD next to your resume. Identify the **top ~5 skills/keywords** the JD repeats.
- [ ] Mirror them into your **summary + skills** using the JD's *exact phrasing* — if the JD says "Spring Cloud Gateway," don't leave it as just "API gateway"; if it says "CI/CD," don't leave it as "DevOps."
- [ ] Verify each company's stack before applying — **don't apply to .NET shops.**
- [ ] Tailor the cover note / application message in 3 sentences: (1) specific relevant project, (2) tech match to the JD, (3) what you want — and drop **"available to join immediately"** in sentence 3. Never send the exact same note twice.

> **ATS routine in one line:** Paste JD → mirror top ~5 keywords (exact phrasing) into summary + skills → verify the stack isn't .NET → 3-sentence tailored note ending in immediate availability → apply → log. Under 5 minutes.

*Applications block (~90 min):*
- [ ] **Apply to 4–5 more**, every one through the routine above. Keep the GCC/product + safety-net mix.
- [ ] Running total target by end of Tuesday: **8–10 applications.**
- [ ] Log every row. Check inbound on Naukri/Instahyre/Cutshort and respond to anything matching your criteria within hours.

*First-screen readiness (~15 min) — CTC + availability scripts:*

The "current/expected CTC" question hits in the *very first* recruiter screen. Have these memorised so you don't anchor low:
- **Deflect (default):** *"I'd rather focus on the role and the total opportunity first — I'm confident we can find a number that works for both sides once I understand the scope."*
- **Researched band if pressed:** *"For Java / Spring Boot engineers with my background in Pune/Bengaluru, the market is around ₹18–24 LPA — I'd want to be in that range."* Name the band, never your current number.
- **Availability as a closer:** *"And I'm available to join immediately — no notice period to serve."* A real lever for teams with an urgent backfill; lead with it whenever timelines come up.

**Pipeline:** Running total at 8–10 applications, every one ATS-tailored and logged; inbound on all four channels triaged within hours.

**EOD check:**
- [ ] Can you ATS-tailor and submit one application in under 5 minutes?
- [ ] Can you deflect the CTC question and pivot to immediate availability in one breath?
