# Full-Time Interview Prep — Day-by-Day Checklists

Companion to [`career-plan-2026.md`](../career-plan-2026.md) (market analysis + strategy) and [`../interview-qa.md`](../interview-qa.md) (Q&A bank).

**Goal:** walk into any interview — GCC, product, or services — able to answer *anything*, including curveballs. Each file has a daily checklist (DSA block + core topic block + self-check), the concepts to master with depth, sample questions including unexpected ones, an "extraordinary-candidate edge" section, and an end-of-week self-assessment gate.

**Cadence:** **full-time from Jun 29** — study weeks (W1–5) run **Mon–Sat, ~36 hr/week** in 3 blocks/day (Block A DSA · Block B core · Block C 2nd DSA/deep-dive), Sunday rest; execution weeks (W6–12) run Mon–Sun. A 1-day Java-fundamentals refresh (Week 0, Sun Jun 28) precedes the sprint. **Study finishes Sat Aug 1; applications run Aug 3 → Sep 20.** Candidate is **available to join immediately** (no notice).

**After Week 12:** offers realistically cluster late-Sept → late-Oct (loops run ~3–6 weeks). With **immediate availability** you can accept and start fast — use it as leverage. Target comp ₹14–25 LPA (40–70% jump realistic).

> **Realism note:** The calendar ends Sep 20, but that's *not* when offers necessarily land — Indian product/GCC loops run ~3–6 weeks application→offer, so offers realistically cluster **late-Sept through late-Oct**; a slip into early October is normal. Two practical rules: (1) From W8 on, a **scheduled interview always overrides the daily template** — never miss an interview to tick a DSA box. (2) Because there's no incoming salary, run a **two-track** application strategy (target GCC/product + faster mid/services safety net) and aim for **~50–70 applications**, so enough loops are live to produce overlapping offers for leverage.

### How to use
1. Each Monday, open that week's file and skim the goal + self-assessment gate first — that's your target.
2. Tick the daily checklist; log what you actually did.
3. Sunday: run the end-of-week self-assessment. **Don't advance if you fail a gate** — repeat the weak day instead. The schedule is a guide, not a deadline.
